O que são Passkeys?

Uma alternativa segura e resistente a phishing às senhas. Baseadas no padrão WebAuthn, usam criptografia de chave pública e permitem autenticação com biometria ou PIN do dispositivo, eliminando a necessidade de senhas tradicionais.

O que é SPC?

Secure Payment Confirmation é uma API web que estende o WebAuthn. Ela foi projetada para autenticar transações, fornecendo prova criptográfica de que o usuário consentiu com os detalhes específicos de um pagamento (valor, beneficiário), e não apenas que fez login.

Considerações de Segurança

A implementação de passkeys e SPC requer atenção a detalhes de segurança críticos para proteger os usuários e as transações. O princípio mais importante é a mitigação de ataques de repetição.

🛡️

Mitigando Ataques de Repetição com `challenge`

O `challenge` é a principal defesa contra ataques de repetição, onde um invasor intercepta uma resposta de autenticação e a reutiliza.

  • Único por Solicitação: O backend DEVE gerar um novo `challenge` criptograficamente seguro para cada tentativa de registro ou autenticação.
  • Validade Curta: O `challenge` deve ser armazenado na sessão do usuário e ter uma vida útil curta (alguns minutos).
  • Uso Único: Uma vez que um `challenge` é usado (seja em uma tentativa bem-sucedida ou falha), ele DEVE ser invalidado imediatamente no backend para que não possa ser usado novamente.

Usar uma biblioteca FIDO no lado do servidor (como SimpleWebAuthn) ajuda a gerenciar esses requisitos de segurança de forma robusta e confiável.
